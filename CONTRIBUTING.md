# Contributing to Awesome AI Orchestration

Thank you for your interest in contributing!

## Guidelines

### Adding a New Tool

1. Make sure the tool is related to AI agent orchestration
2. Add it to the appropriate category in `README.md`
3. Follow the existing table format
4. Include a working GitHub link and brief description
5. The tool should have at least 50+ GitHub stars (exceptions for very new but notable tools)

### Format

```markdown
| [Tool Name](https://github.com/owner/repo) | Brief description | ![GitHub stars](https://img.shields.io/github/stars/owner/repo) |
```

### Categories

- **Protocols & Standards** — Open protocols for agent interoperability
- **Multi-Agent Frameworks** — Libraries for building multi-agent systems
- **Coding Agent Orchestrators** — Tools for running coding agents in parallel
- **Low-Code / Visual Platforms** — Visual workflow builders
- **Observability & Evaluation** — Monitoring and debugging tools
- **Agent Infrastructure** — Execution environments and sandboxes
- **Personal AI Assistants** — Always-on AI assistants
- **Autonomous Loop Runners** — "Run until done" patterns
- **Multi-Agent Swarms** — Coordinated agent swarms

### Pull Request Process

1. Fork the repository
2. Create a new branch for your addition
3. Add your entry in the correct alphabetical position within its category
4. Submit a pull request with a clear description of what you're adding and why

### Quality Standards

- No duplicates
- No broken links
- Tools must be actively maintained (commit within last 6 months)
- No promotional or paid-only tools without clear value

## Code of Conduct

Be respectful and constructive in all interactions.
